# Typora Theme Gallery taxonomy for Shadow Engine

The gallery shows a large ecosystem of themes. Instead of importing everything, Shadow should classify by intent.

## Observed theme families

| Family | Examples from gallery | Shadow use |
|---|---|---|
| AI/tool inspired | OpenCode, Claude-Like, Vercel, Warp Gradient | model/product review surfaces |
| code/dark IDE | JetBrains Dark, OneDark, Dracula, Nord, GitHub Night, Compact Night | focus/watch/code modes |
| editorial/print | Print, Paper, Newsprint, Academic, LaTeX Typora, NovelTex | calm/publish/long-read modes |
| note/productivity | Notion, Notion Style, Bear, Lark, Fluent | work notes and clean docs |
| cyber/neon | Cyberpunk, Neon, Phycat-neon, Blackout | urgent/watch/high-signal surfaces |
| color mood | Rose, Purple, Mint, Lavender, Saffron, OrangeHeart | mode accents |
| branded/platform mimicry | GitHub, GitLab, Vercel, Dropbox, Vue, Tailwind | target-surface mimicry |
| specialized writing | Screenplay, Law, Middle East, Uyghur, Han | domain-specific output modes |
| configurable/framework | Typora Themeable, VLOOK series | engine inspiration: composable theme systems |

## Shadow design takeaway

The gallery is not just a download list. It is a latent taxonomy of document states:

```text
code / editorial / academic / brand / product / neon / minimal / legal / note / configurable
```

Shadow Typora Engine should route by **document state**, not by aesthetic preference.
